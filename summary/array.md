# 数组

## 基础知识

```shell
    1. 线性表（Linear List）,每个线性表上的数据最多只有两个方向(一前一后)，例如，数组，链表，队列，栈
    2. 非线性表, 数据之间并不是简单的前后关系, 例如，树，堆，图等
    3. 数组支持随机访问，根据下标随机访问的时间复杂度为 O(1),主要是通过基地址和偏移量取数据,
       更底层的原理在实现上使用的是连续的内存空间，可以借助 CPU 的缓存机制，预读数组中的数据，所以访问效率更高．
       因为内存的局部性原理，当访问数组中某个元素时，CPU 会从内存中读取一个数据块到 CPU 缓存中，访问　CPU 缓存的
       速度要远快于访问内存
    
```

## 数组的特点

```shell
    1. 线性表, 数据的方向只有前后关系
    2. 连续的内存空间和相同类型的数据
            这个特点将导致随机访问，同时正因为具备连续的存储空间和相同的数据类型，当删除和插入时，
            必将导致大量的数据搬移
    
    3. 优点： 访问效率高
    　　缺点: 申请的大小固定
```

## 数组操作优化

```shell
    1. 插入操作
            由于数组保存的数据的连续性，导致从中插入一个值，会使得这个位置的后面所有的元素向后移一个．
            优化策略:
                (1) 如果这个数组是无序的，存储的数据并没有任何规律，只是当作存储数据的集合，则可以将第 k 位的数据搬移到
                　　数组元素的最后，把新的元素直接放入第 k 个位置。
                (2) 如果这个数组是有序的，那没有办法
                
    2. 删除操作
            常规流程是要删除 a[k], 则将 k 之后的元素统一前移一个位置．
            优化策略:
                    可以先记录已经删除的数据,每次的删除操作并不是真正地搬移数据，只是标记该数据已经被删除。
                    当数组没有更多空间存储数据时，再触发执行一次真正的删除操作，减少删除操作导致的数据搬移．
                    
```
